# 矩阵运算 算法替代参考

供 `perspective-algorithm`（算法模式识别与替代研究员）检测时 Read 引用。聚焦**算法/数学层面**的改进方案，不含指令级优化。

---

## 1. 通用矩阵乘（GEMM）

| 当前方案 | 替代方案 | 复杂度变化 | ARM 适配 |
|---------|---------|-----------|---------|
| 朴素 3 层嵌套 `C[i][j] += A[i][k] * B[k][j]` | 分块 (block/tile) + 寄存器打包 | O(N³) → O(N³)，常数因子 ↓ 10-100× | 适配 L1/L2 cache 大小分块 |
| 已分块 | 微内核 (micro-kernel) 展开 4×4/8×8 | O(N³)，常数因子 ↓ 2-4× | NEON FMLA 4×4 微内核 |
| 微内核 | 更激进的 packing 格式（如列主序→行主序重排） | O(N³)，常数因子 ↓ | SVE gather load 可免 packing |

**ARM 关键参数**：
- Kunpeng-0xd01: L1d 64KB → 分块大小建议 ≤ 48KB（留余量给栈/其他变量）
- NEON: 32 个 128-bit 寄存器 → 微内核最大 8×12（128 寄存器限制）
- SVE: 32 个可扩展寄存器 → 微内核尺寸随 VL 缩放

**检测信号**：
- 3 层嵌套 for 循环，内层为 `C[i*ldc+j] += A[i*lda+k] * B[k*ldb+j]`

## 2. 三角求解 / 分解

| 当前方案 | 替代方案 | 复杂度变化 | ARM 适配 |
|---------|---------|-----------|---------|
| 朴素前向/后向代入 | 分块三角求解 | O(N²) → O(N²)，常数因子 ↓ | 分块 BLAS 标准实现 |
| LU/Cholesky（标量） | 分块 LU/Cholesky | O(N³) → O(N³)，常数因子 ↓ | LAPACK/OpenBLAS 已实现 |
| 小矩阵 LU（≤ 16×16）| 展开 + NEON 向量化 | O(N³)，常数因子 ↓ 5-10× | NEON FMLA 密集 |

## 3. 稀疏矩阵

| 当前方案 | 替代方案 | 复杂度变化 | ARM 适配 |
|---------|---------|-----------|---------|
| COO 格式 | CSR/CSC（压缩稀疏行/列） | SpMV 访存 ↓ 3× | 标准移植 |
| CSR SpMV（标量） | CSR SpMV NEON 向量化 | O(Nnz)，常数因子 ↓ 2-4× | NEON 4-lane 同时处理 4 个非零元 |
| 稠密化稀疏矩阵（Dense 存储） | 保持稀疏格式 | 内存 ↓ 10-100× | 当稀疏度 > 90% 时 |

**检测信号**：
- SpMV: `for(i) for(jj = row_ptr[i]; jj < row_ptr[i+1]; jj++) y[i] += val[jj] * x[col_idx[jj]]`
