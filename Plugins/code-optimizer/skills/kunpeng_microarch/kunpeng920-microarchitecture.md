## 1. 流水线总览

```
         In-Order                                          Out-of-Order
┌───────┐  ┌────────┐  ┌──────────┐  ┌──────────┐  ┌───────┐  ┌──────────┐         ┌───────┐  ┌────────┐
│ Fetch │─►│ Decode │─►│ Allocate │─►│ Dispatch │─►│ Issue │─►│ ALU (P0) │         │ Write │─►│ Commit │
│ 4-wide│  │ 4-wide │  │ /Rename  │  │          │  │       │  ├──────────┤         │ Back  │  │ 4-wide │
└───────┘  └────────┘  │  ROB~96  │  └──────────┘  └───────┘  │ALU+BR(P1)│         └───────┘  └────────┘
                       │  PRF     │                            │ALU+BR(P2)│             ▲
                       └──────────┘                            ├──────────┤             │
                                                               │ MDU (P3) │─────────────┘
                                                               ├──────────┤
                                                               │FP/SIMD(P4│  LDQ/STQ
                                                               │     P5)  │      │
                                                               ├──────────┤      ▼
                                                               │LD/ST (P6)│  ┌───────┐
                                                               │     P7)  │  │ L1D $ │──► L2 $ ──► L3 $ ──► DDR4
                                                               └──────────┘  │64K 4w │   512K     Ring     4ch
                                                                             └───────┘
```

| 阶段 | 宽度/容量 | 说明 |
|------|----------|------|
| Fetch | 4 指令/cycle | L1I 命中时 |
| Decode | 4-wide | 含 Move Elimination |
| Rename/Allocate | ROB ~96 entry | Goldmont Plus 量级 |
| Issue | ≤4 uop/cycle | 3 个统一调度器，各 ~33 entry |
| Commit | 4 uop/cycle | LLVM AArch64SchedTSV110.td |

## 2. 前端：取指单元

### 2.1 取指 (Instruction Fetch)

| 参数 | 数值 | 备注 |
|------|------|------|
| 取指宽度 | 4 指令/cycle | L1I 命中时 |
| L1 I-Cache 容量 | 64 KB | |
| L2 取指延迟 | 11–12 cycle | 每 64B cacheline 一个分支时 |
| L3 取指延迟 | >38 cycle | 极差，接近 DRAM 级别 |

> ⚠️ 分支预测器无法超前于取指流水线驱动预取，这是 L3 取指性能极差的根本原因。

### 2.2 分支预测器

| 参数 | 数值 | 备注 |
|------|------|------|
| 预测器类型 | 两级动态分支预测器 | 具体算法未披露 |
| 一级 BTB | 64 entry | 单周期跳转目标，零气泡 Taken Branch |
| 二级覆盖范围 | 32 KB 代码内 3 cycle 处理 | |
| 分支间距限制 | ≤16B 间距：+1 cycle 惩罚 | 更密集则更差 |
| 返回地址栈 (RAS) | 31 entry | |
| 间接分支预测 | 最多 16 历史周期，~256 间接目标 | 超出后显著惩罚 |
| 行为参考 | 与 Cortex A73 行为相近 | 随机 taken/not-taken 模式测试 |

### 2.3 译码单元

| 参数 | 数值 | 备注 |
|------|------|------|
| 译码宽度 | 4-wide | 每周期 4 条指令 → micro-op |
| Move Elimination | 支持 | 在寄存器重命名阶段执行 |

### 2.4 前端性能瓶颈

| 问题 | 影响 | 缓解方向 |
|------|------|---------|
| L3 取指 >38 cycle | 大代码段/ICache miss 代价极高 | 热路径紧凑布局、避免代码膨胀 |
| 分支预测器无预取能力 | L2/L3 取指延迟直接暴露 | 减少 ICache miss |
| 高密度分支 ≤16B 惩罚 | 代码布局敏感 | 热路径分支间距 >16B |
| 间接目标仅 ~256 | 虚函数密集代码受影响 | 去虚拟化、switch→if-else |

## 3. 重命名/分配

### 3.1 ROB

| 参数 | 数值 | 备注 |
|------|------|------|
| ROB 容量量级 | ~96 entry | 与 Goldmont Plus 相近 |
| 实际压力 | 🟡 中 | 调度器通常先于 ROB 耗尽 |

### 3.2 物理寄存器堆 (PRF)

| 参数 | 数值 | 备注 |
|------|------|------|
| 整数 PRF 压力 | 🟢 低 | SPEC CPU2017 极少耗尽 |
| FP/向量 PRF 压力 | 🟠 中高 | 浮点负载下显著承压 |

> ⚠️ **关键设计缺陷**：AArch64 定义 32 个架构 FP/向量寄存器（x86-64 仅 16），相同 PRF 容量下可重命名槽更少。浮点密集代码应减少活变量以降低 PRF 压力。

## 4. 发射/执行

### 4.1 发射宽度与调度器

| 参数 | 数值 | 备注 |
|------|------|------|
TaiShan v110 采用 **分类统一调度 (Clustered Unified Scheduler)**：

| 调度器 | 条目数 | 服务端口 | 压力等级 |
|--------|--------|---------|---------|
| ALU 调度器 | ~33 | 整数执行 ×4 | 🔴 高 |
| 内存调度器 | ~33 | AGU ×2 | 🔴 高 |
| FP/向量调度器 | ~33 | FPU ×2 | 🔴 高 |

> 对比：Goldmont Plus 整数侧为分布式调度器，Neoverse N1 为纯分布式。统一调度器灵活性更好，但跨类型无法共享条目。

### 4.2 整数执行端口

| 端口 | 功能 | 延迟 |
|------|------|------|
| Port 0 | 通用 ALU（不支持分支） | 1 cycle |
| Port 1 | 通用 ALU + Taken Branch | 1 cycle |
| Port 2 | 通用 ALU + Taken Branch | 1 cycle |
| Port 3 | 整数乘法/除法专用 | mult 4c, div 多周期 |

> ⚠️ 分支放在 Port 1/2（通用 ALU 端口）而非独立端口，Goldmont Plus / Neoverse N1 则将分支放第 4 端口以优先处理、更快发现误预测。

**OSACA 端口模型 (LLVM AArch64SchedTSV110.td)**：

```
+--------------------------------------------------------------------------------------------+
|                                - entries                                                   |
+--------------------------------------------------------------------------------------------+
  0 |ALU      1 |AB       2 |AB       3 |MDU           4 |FSU1   5 |FSU2    6 |LdSt   7 |LdSt
    \/          \/          \/          \/               \/        \/         \/        \/
 +---------+ +---------+ +---------+ +-------------+  +-------+ +------ +  +-------+ +-------+
 | INT ALU | | INT ALU | | INT ALU | | Multi-Cycle |  |   FP  | |   FP  |  | LD/ST | | LD/ST |
 +---------+ |   BRU   | |   BRU   | +-------------+  | ASIMD | | ASIMD |  +-------+ +-------+
             +---------+ +---------+                  +-------+ +-------+
```

### 4.3 浮点/向量执行端口

| 端口 | 功能 | 向量宽度 |
|------|------|---------|
| FP Port 0 | FMA(FP32/FP64), FP32 ADD, 向量整数 ADD/MUL | 128-bit |
| FP Port 1 | FMA(FP32/FP64), FP32 MUL, 向量整数 ADD | 128-bit |

> ⚠️ FP32 ADD 仅 Port 0，FP32 MUL 仅 Port 1（延迟相同 5 cycle），限制混合 ADD/MUL 吞吐。

### 4.4 内存执行端口

| 端口 | 功能 |
|------|------|
| AGU 0 | Load 或 Store |
| AGU 1 | Load 或 Store |

每周期最多：2 Load 或 1 Load + 1 Store

### 4.5 指令延迟与吞吐量汇总

| 操作 | 延迟 | 吞吐量 (per cycle) | 向量宽度 | 端口 |
|------|------|-------------------|---------|------|
| 整数 ALU | 1c | 3 | — | P0/P1/P2 |
| 整数乘法 | 4c | 1 | — | P3 |
| 整数除法 | 多周期 | <1 | — | P3 |
| Taken Branch | — | 1 | — | P1/P2 |
| FP32 FMA | 5c | 2 | 128-bit | FP0+FP1 |
| FP64 FMA | 5c(推测) | 0.5 | 128-bit | 四分之一速率 |
| FP32 ADD | 5c | 1 | 128-bit | 仅 FP0 |
| FP32 MUL | 5c | 1 | 128-bit | 仅 FP1 |
| 向量整数 ADD | 2c | 2 | 128-bit | FP0+FP1 |
| 向量整数 MUL | — | 1 | 128-bit | 仅 FP0 |
| L1D Load (简单寻址) | 4c | 2 | — | AGU×2 |
| L1D Load (索引寻址) | 5–6c | 2 | — | AGU×2 |
| Store Forwarding | 6–7c | — | — | — |
| Store Fwd (跨 16B) | 7–9c | — | — | +1~2c 惩罚 |
| L2 Load | 10c | — | — | — |
| L2 TLB Miss | +11c | — | — | — |
| L3 Load (Partition 近端) | ~36c | — | — | — |
| L3 Load (Shared/远端) | >90c | — | — | — |
| DRAM (空载) | 96 ns | — | — | — |

## 5. 访存子系统

### 5.1 Load Queue / Store Queue

| 参数 | 数值 | 备注 |
|------|------|------|
| Store Forwarding | 6–7c | 部分重叠转发延迟相同 |
| 跨 16B 边界惩罚 | +1~2c | L1D 以 16B 对齐块操作 |
| Load/Store 并行条件 | 不跨 16B 边界可并行 | 否则需串行 |

### 5.2 TLB 层次

| 层级 | 条目数 | 相联度 | 命中延迟 | 备注 |
|------|--------|--------|---------|------|
| L1 I-TLB | 32 | 全相联 | — | |
| L1 D-TLB | 32 | 全相联 | — | miss 后查 L2 TLB |
| L2 Unified TLB | 1024 | — | +11c | 可能指令/数据共享 |

> ⚠️ L2 TLB 延迟 +11c 慢于 Zen 2 (7c, 2048 entry) 和 Goldmont Plus (8c)。容量大但慢，TLB miss 代价高，大页 (2MB/1GB) 收益明显。

### 5.3 L1 Data Cache

| 参数 | 数值 |
|------|------|
| 容量 | 64 KB |
| 组相联度 | 4-way |
| 访问延迟 | 4c (简单寻址), 5–6c (索引寻址) |
| 读取带宽 | 2 × 128-bit/cycle |
| 操作粒度 | 16B 对齐块 |

### 5.4 L2 Cache

| 参数 | 数值 |
|------|------|
| 容量 | 512 KB/core (私有) |
| 访问延迟 | 10c (优于 Neoverse N1 和 Zen 2) |
| L2→L1D 带宽 | ~32 B/cycle 接口 (实测 ~20 B/cycle) |
| 接口方向 | 单向 |

### 5.5 L3 Cache

| 参数 | 数值 |
|------|------|
| 物理分布 | 每 Cluster 对应一个 L3 Bank，独立 Ring Stop |
| Tag 位置 | 位于 CPU Cluster (非 L3 Bank)，与 Intel/AMD 不同 |

**L3 工作模式**：

| 模式 | 触发条件 | 延迟 |
|------|---------|------|
| Partition 近端 | 单核访问，数据 < ~4 MB | ~36c |
| Partition 远端扩展 | 数据量逐渐增大 | 36 → 90+c |
| Shared | 多核共享 / 数据量接近 L3 容量 | >90c |

**L3 带宽**：

| 场景 | 带宽 |
|------|------|
| 单 Cluster (4核) 读取 | 21.7 GB/s |
| 带宽竞争时延迟 | >80 ns |

> ⚠️ L3 Partition 近端 ~36c vs Shared >90c，3 倍差值是 Kunpeng-0xd01 最重要的访存优化规则。数据布局要严格控制共享范围，避免触发 Shared 模式。

### 5.6 DRAM 子系统

| 参数 | 数值 |
|------|------|
| 控制器位置 | 计算芯片顶部 + 底部边缘 |
| 通道配置 | 双通道 ×2 组 = 4 通道 |
| 内存类型 | DDR4-2400 |
| 读取带宽 (纯读) | 63 GB/s |
| 空载延迟 | 96 ns |
| 中等负载延迟 | >100 ns |
| 满载延迟 | >300 ns |

### 5.7 硬件预取器 & MSHR

| 参数 | 数值 |
|------|------|
| 硬件预取器类型 | SEQ（流式内存访问预取器），其余类型未披露 |

### 5.8 访存子系统层次总览

```
AGU × 2 (地址生成，4c L1D hit)
    │
    ├─► L1 D-TLB (32 entry, fully-assoc)
    │       └─► L2 Unified TLB (1024 entry, +11c) ──► Page Table Walker
    │
    ▼
L1 D-Cache (64 KB, 4-way, 4c, 2×128-bit/cycle, 16B 对齐块)
    │  Store Forwarding: 6–7c (跨 16B 边界 +1~2c)
    ▼
L2 Cache (512 KB/core, 私有, 10c, ~32B/cycle 接口, 单向)
    │
    ▼
L3 Cache (Bank per Cluster, Ring Bus 互联, Tag 在 Cluster 侧)
    │  Partition 模式: 36~90+c
    │  Shared 模式:    >90c
    │  Cluster 带宽:   21.7 GB/s
    ▼
DDR4-2400 (4 通道, 63 GB/s, 96ns 空载, >300ns 满载)
```

## 9. 优化要诀

1. **控制 L3 Shared 模式触发**：多核共享数据布局需隔离，单核工作集 <4 MB 保持 Partition 近端 ~36c
2. **16B 对齐**：Store Forwarding 跨 16B 边界仅 +1~2c，对齐门槛不高但需遵守
3. **大页优先**：L2 TLB +11c 代价高，2MB/1GB 大页收益显著
4. **分支间距 >16B**：高密度分支引入额外惩罚
5. **FP 代码减少活变量**：缓解 FP/向量 PRF 压力
6. **FP32 ADD/MUL 交错排布**：避免同类型指令扎堆堵塞单端口
7. **跨 NUMA 显式处理**：每 SCCL 独立 NUMA 节点，无透明共享
8. **代码紧凑布局**：降低 ICache miss，避免 L3 取指 >38c
